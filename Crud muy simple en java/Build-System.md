## Purpose and Scope

This document describes the Apache Ant-based build system used to compile, test, and package the `crud3` student management application. It covers the build configuration files, build lifecycle, project structure, and dependency management.

For information about running the compiled application, see [Running the Application](/axchisan/Crud-MUUUy-simple-en-java-de-hace-a-os/2.3-running-the-application). For details about the NetBeans Form files that are processed during compilation, see [NetBeans Form Definitions](/axchisan/Crud-MUUUy-simple-en-java-de-hace-a-os/8.1-netbeans-form-definitions-(.form-files)).

## Build System Overview

The `crud3` project uses **Apache Ant** as its build automation tool, managed through **NetBeans IDE**. The build system follows a two-layer architecture:

* **User-editable layer**: [build.xml L1-L74](https://github.com/axchisan/Crud-MUUUy-simple-en-java-de-hace-a-os/blob/7ec3bd78/build.xml#L1-L74)  provides customization hooks and imports the generated implementation
* **Generated layer**: [nbproject/build-impl.xml L1-L4500](https://github.com/axchisan/Crud-MUUUy-simple-en-java-de-hace-a-os/blob/7ec3bd78/nbproject/build-impl.xml#L1-L4500)  contains the complete build logic generated by NetBeans

This separation allows developers to customize the build process without modifying auto-generated code. The build system compiles Java sources from `src/`, packages them into an executable JAR at `dist/crud3.jar`, and manages external dependencies like the MySQL JDBC driver.

**Sources**: [build.xml L1-L74](https://github.com/axchisan/Crud-MUUUy-simple-en-java-de-hace-a-os/blob/7ec3bd78/build.xml#L1-L74)

## Summary

The `crud3` build system provides a complete Ant-based build automation solution managed by NetBeans IDE:

* **Two-layer architecture**: User-editable [build.xml L1-L74](https://github.com/axchisan/Crud-MUUUy-simple-en-java-de-hace-a-os/blob/7ec3bd78/build.xml#L1-L74)  imports generated [nbproject/build-impl.xml L1-L4500](https://github.com/axchisan/Crud-MUUUy-simple-en-java-de-hace-a-os/blob/7ec3bd78/nbproject/build-impl.xml#L1-L4500)
* **Standard Java build lifecycle**: init → compile → test → jar → javadoc
* **Customization hooks**: Pre/post hooks for each build phase ([build.xml L13-L72](https://github.com/axchisan/Crud-MUUUy-simple-en-java-de-hace-a-os/blob/7ec3bd78/build.xml#L13-L72) )
* **Dependency management**: MySQL Connector/J packaged with application via CopyLibs
* **Executable JAR**: `dist/crud3.jar` with `GUI.GUI` as entry point
* **Java 21 compilation**: Modern Java version with full feature support
