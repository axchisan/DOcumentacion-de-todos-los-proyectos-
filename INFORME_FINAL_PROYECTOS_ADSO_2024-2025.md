### 4.2 CRUD de Estudiantes en Java (Octubre 17, 2024)

#### 4.2.1 Información Básica
- **Nombre del Proyecto:** Sistema de Registro de Estudiantes
- **Fecha de Desarrollo:** Octubre 17, 2024
- **Instructor:** Alexander Díaz
- **Equipo de Desarrollo:** Duvan Yair Arciniegas Gerena, Gerardo Andrés Ardila Jerez
- **Objetivo General:** Desarrollar un sistema de gestión de estudiantes con operaciones CRUD completas y persistencia en base de datos.

#### 4.2.2 Contexto y Requisitos
Proyecto diseñado para aprender los fundamentos de las operaciones CRUD (Create, Read, Update, Delete) en aplicaciones de escritorio con conexión a base de datos. Se requería implementar un sistema completo de gestión de información de estudiantes con interfaz gráfica y almacenamiento persistente.

#### 4.2.3 Herramientas y Tecnologías Utilizadas
- **Lenguaje de Programación:** Java
- **Framework de UI:** Java Swing con NetBeans Form Designer
- **Base de Datos:** MySQL
- **Conectividad:** JDBC (Java Database Connectivity)
- **IDE:** NetBeans
- **Sistema de Construcción:** Apache Ant

#### 4.2.4 Arquitectura del Sistema
El proyecto implementa una arquitectura en tres capas bien definida:

**Capa de Presentación (UI Layer):**
- `GUI.GUI`: Formulario principal de entrada de datos
- `AdminForm`: Interfaz de administración
- `ComponentePaneles`: Componentes reutilizables de interfaz

**Capa de Lógica de Aplicación (Application Logic Layer):**
- `alumno`: Modelo de datos del estudiante
- `alumnoDataChange`: Servicio de gestión de cambios
- `ValidadarDatos`: Validación de entrada de datos
- `LimpiarCampos`: Gestión de campos del formulario

**Capa de Acceso a Datos (Data Access Layer):**
- `conexionDB`: Gestión de conexión a base de datos
- Operaciones CRUD implementadas con PreparedStatements

#### 4.2.5 Esquema de Base de Datos
```sql
CREATE TABLE estudiantes (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nombre VARCHAR(100) NOT NULL,
    apellido VARCHAR(100) NOT NULL,
    identificacion VARCHAR(20) UNIQUE NOT NULL,
    programa VARCHAR(100),
    ficha VARCHAR(20),
    fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.2.6 Funcionalidades Implementadas
- **Crear Estudiante:** Registro de nuevos estudiantes con validación de datos
- **Leer Estudiantes:** Visualización de lista completa con búsqueda
- **Actualizar Estudiante:** Modificación de información existente
- **Eliminar Estudiante:** Eliminación con confirmación
- **Validación de Datos:** Verificación de campos obligatorios y formatos
- **Gestión de Formularios:** Limpieza automática de campos

#### 4.2.7 Flujo de Datos
1. Usuario ingresa datos en el formulario GUI
2. `ValidadarDatos` verifica la integridad de la información
3. `alumnoDataChange` procesa la operación solicitada
4. `conexionDB` ejecuta la consulta SQL correspondiente
5. Base de datos persiste los cambios
6. Interfaz se actualiza con los resultados

#### 4.2.8 Resultados
- Sistema CRUD completamente funcional
- Interfaz gráfica intuitiva con validación en tiempo real
- Conexión estable a base de datos MySQL
- Manejo adecuado de excepciones y errores
- Código organizado siguiendo principios de separación de responsabilidades

#### 4.2.9 Desafíos y Soluciones
**Desafío:** Gestión eficiente de conexiones a base de datos.
**Solución:** Implementación de patrón Singleton para la conexión y uso de PreparedStatements para prevenir SQL injection.

**Desafío:** Validación de datos antes de insertar en base de datos.
**Solución:** Creación de clase `ValidadarDatos` con métodos específicos para cada tipo de campo.

**Desafío:** Sincronización entre interfaz y base de datos.
**Solución:** Implementación de listeners y actualización automática de la vista después de cada operación.

#### 4.2.10 Lecciones Aprendidas
- Importancia de la arquitectura en capas para mantenibilidad del código
- Uso de PreparedStatements para seguridad en consultas SQL
- Validación de datos como práctica esencial en aplicaciones empresariales
- Gestión adecuada de recursos de base de datos (conexiones, statements, resultsets)

#### 4.3.1 Información Básica
- **Nombre del Proyecto:** Haunted Hollow
- **Fecha de Desarrollo:** Noviembre 5, 2024
- **Instructor:** Franklin Alexis Fonseca
- **Equipo de Desarrollo:**

  - José Rojas (Historia y trasfondo)

- **Objetivo General:** Desarrollar un videojuego educativo que mejore el aprendizaje del idioma inglés mediante una experiencia interactiva y divertida.

#### 4.3.2 Contexto y Requisitos
El proyecto fue solicitado para el programa de bilingüismo del SENA, con el objetivo de crear una herramienta educativa innovadora que superara las limitaciones de los métodos tradicionales de enseñanza de inglés (plantillas de texto e hipervínculos). Se requería:
- Integración de mecánicas de juego con aprendizaje de inglés
- Diseño visual atractivo y profesional
- Experiencia de juego envolvente
- Contenido educativo integrado naturalmente en la narrativa

#### 4.3.3 Herramientas y Tecnologías Utilizadas
- **Motor de Juego:** GameMaker Studio
- **Lenguaje de Programación:** GML (GameMaker Language)
- **Diseño Gráfico:** Software de edición de sprites
- **Producción de Audio:** Software de composición musical
- **Tiempo de Desarrollo:** 2 meses de trabajo intensivo

#### 4.3.4 Concepto del Juego
Haunted Hollow es un juego de aventura y acertijos ambientado en mazmorras encantadas. El jugador encarna a un mago atrapado que debe enfrentar fantasmas, enemigos y resolver acertijos diseñados para mejorar la comprensión del inglés mientras navega por laberintos desafiantes.

#### 4.3.5 Mecánicas de Juego
- **Exploración:** Navegación por mazmorras con diseño de niveles progresivo
- **Combate:** Sistema de enfrentamiento contra enemigos
- **Acertijos:** Puzzles basados en comprensión de inglés
- **Progresión:** Sistema de avance por niveles
- **Narrativa:** Historia integrada que motiva el aprendizaje

**Diseño Visual (Gerardo Ardila):**
- Sprites de personajes y enemigos
- Diseño de escenarios y ambientes
- Elementos de interfaz de usuario
- Efectos visuales y animaciones

**Programación (Duvan Arciniegas):**
- Lógica de juego en GML
- Sistema de colisiones
- Inteligencia artificial de enemigos
- Gestión de estados del juego
- Sistema de guardado y carga

**Narrativa (José Rojas):**
- Historia principal del juego
- Diálogos en inglés
- Trasfondo de personajes
- Integración de contenido educativo

**Audio (Joxa):**
- Música de fondo
- Efectos de sonido
- Ambiente sonoro

#### 4.3.7 Resultados
- Videojuego completamente funcional y jugable
- Integración exitosa de mecánicas de juego con aprendizaje de inglés
- Experiencia de usuario envolvente y educativa
- Superación de expectativas del proyecto original

**Desafío:** Balanceo entre dificultad del juego y nivel de inglés.
**Solución:** Diseño de niveles progresivos que aumentan gradualmente la complejidad lingüística.
